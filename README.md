# DaVinci — Google-integrated Edge/Cloud Assistant

Node + TypeScript server. Multi-agent brain (router → planner/executor/researcher →
synthesizer), Google login + Gmail/Calendar/Drive adapters, semantic recall over Drive
docs, camera **Vision**, and realtime **Live** voice (WebSocket). Everything is served
same-origin: one HTTP server hosts the API, the static `public/` UI, and the Live
WebSocket.

## Layout

```
src/            server + agents + adapters + stores (TypeScript source of truth)
client/         browser TS for Vision + Live (bundled by esbuild into public/)
public/         index.html, privacy/terms, audio worklets  (client bundles are built)
scripts/dev.mjs local dev runner
.env.example    full config template — copy to .env and fill in
Dockerfile      2-stage build → runtime image (used by Cloud Run / Render / anything)
```

Nothing is pre-built in the repo. `dist/` and the `public/*-client.js` bundles are
generated by `npm run build` and are git-ignored on purpose.

## Run locally

```bash
npm install
cp .env.example .env      # fill in keys (see below)
npm run dev               # tsx watch on the server
# or a production-style run:
npm run build && npm start
```

With no Google/Firebase creds the store falls back to file-backed JSON under `data/`
(dev only). In production it uses **Firestore** automatically.

## Configuration

All config is env vars — see `.env.example` for the annotated list. Minimum to boot:

| Var | Purpose |
|-----|---------|
| `ANTHROPIC_API_KEY` | agents + router + vision (Claude) |
| `GEMINI_API_KEY` | Live voice (required for voice); brain/embedding fallback |
| `OPENAI_API_KEY` | Drive recall embeddings (falls back to Gemini if unset) |
| `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET` | login + Gmail/Calendar/Drive |
| `GOOGLE_REDIRECT_URI` | must exactly match the OAuth console entry |
| `STORE_BACKEND` | `firestore` in prod, `file` for local dev |

**Never commit real keys or `firebase-credentials.json`.** On Google Cloud Run the
runtime service account supplies Firestore access automatically (ADC) — no key file
needed.
